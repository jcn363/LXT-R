**File:** `/home/user/Desktop/LTX-R/Cargo.toml`

### 1b. Create shared test helper crate `ltx-test-utils`

Create `crates/ltx-test-utils/` with:

- `src/lib.rs` — re-exports helpers
- `src/golden.rs` — `load_golden(path, name) -> Tensor` loads a named array from `.npz` and converts to `tch::Tensor`
- `src/assert.rs` — `assert_allclose(a, b, rtol, atol)` compares two tensors element-wise within tolerance
- `src/fixtures.rs` — `make_vs()`, `make_seed_tensor()`, deterministic RNG helpers

**Files:**

## Task 2: Bug Verification Tests

Targeted tests that prove each of the 4 bugs is actually fixed — not just shape-correct but numerically correct.

### 2a. TransformerAttention: multi-head with `head_dim != dim`

**File:** `crates/ltx-attention/tests/transformer_attn.rs` (add tests)

Tests to add:
- `test_transformer_attention_multi_head` — `dim=64, heads=4, head_dim=16`, verify output shape `[B, seq, 64]` and output is non-zero
- `test_transformer_attention_multi_head_with_rope` — same config with RoPE applied, verify output shape

### 2b. Block: modulation broadcasts uniformly across sequence

**File:** `crates/ltx-transformer/tests/block.rs` (add tests)

Tests to add:
- `test_block_modulation_uniform_across_seq` — create block, feed same batch with `seq=1` and `seq=10`, verify outputs at each position are identical (proves modulation broadcasts, not tiling)
- `test_block_large_batch_large_seq` — `batch=4, seq=20, dim=64`, stress test

### 2c. BlurDownsample: verify kernel values

**File:** `crates/ltx-upsampler/tests/blur_downsample.rs` (add tests)

Tests to add:
- `test_blur_kernel_shape_and_sum` — expose `make_gaussian_kernel` (make it `pub(crate)` + `#[doc(hidden)]` or test via `BlurDownsample` internals), verify kernel is `[K, K]` and sums to ~1.0
- `test_blur_kernel_symmetry` — verify `K[i,j] == K[K-1-i, K-1-j]` (central symmetry)
- `test_blur_kernel_center_is_max` — verify center element is the largest

### 2d. FP8Linear: non-square weight numerical correctness

**File:** `crates/ltx-quantization/tests/fp8_mm.rs` (add tests)

Tests to add:
- `test_fp8_linear_non_square` — `weight=[8, 32]`, `x=[1, 32]`, verify output shape `[1, 8]` and non-zero
- `test_fp8_linear_matches_reference` — manually dequantize weight, compute `x @ w.t()` in f32, compare against FP8Linear output within quantization tolerance (~1.0 max error for FP8)
- `test_fp8_linear_numerical_identity` — use identity-like weight, verify `x @ I^T ≈ x`
