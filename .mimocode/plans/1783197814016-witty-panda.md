# Plan: T5 Text Encoder for Prompt Conditioning

## Goal
Add T5EncoderModel support so prompts actually produce meaningful text embeddings for the transformer's cross-attention layer. The LTX-Video 2B model uses T5-XXL (not Gemma3+SigLIP).

## T5 Architecture (from checkpoint config)

| Param | Value |
|-------|-------|
| d_model | 4096 |
| d_ff | 10240 |
| d_kv (head_dim) | 64 |
| num_heads | 64 |

| Total weights | ~18 GB (219 tensors) |

## Key Differences from Gemma3

| Aspect | Gemma3 (current) | T5 (needed) |
|--------|-----------------|-------------|
| Position encoding | RoPE | Relative attention bias |
| Normalization | RMSNorm | LayerNorm |
| FFN | SwiGLU | Gated GELU |
| Attention | GQA (16Q/8KV heads) | MHA (64Q/64KV heads) |
| QK norm | Yes | No |
| Dropout | 0.0 | 0.1 |
| hidden_size | 3840 | 4096 |
| Output context_dim | 3840 | 4096 |

## Implementation Details

### T5ConfigData
```rust
#[derive(Deserialize)]
pub struct T5ConfigData {
    pub d_model: i64,           // 4096
    pub d_ff: i64,              // 10240
    pub d_kv: i64,              // 64 (head_dim)
    pub num_heads: i64,         // 64
    pub num_layers: i64,        // 24
    pub vocab_size: i64,        // 32128
    pub layer_norm_epsilon: f64, // 1e-6
    pub dropout_rate: f64,      // 0.1
    pub relative_attention_num_buckets: i64,  // 32
    pub relative_attention_max_distance: i64, // 128
    pub is_gated_act: bool,     // true
    pub dense_act_fn: String,   // "gelu_new"
}
```

### T5RelativePositionBias
- Learnable bias table: `[num_buckets, num_heads]` = `[32, 64]`
- Shared across all 24 layers (only in block.0 checkpoint)
- Bucket assignment: relative_position ‚Üí bucket index via logarithmic bucketing
- Added to attention scores before softmax

### T5LayerNorm
- Standard LayerNorm with weight `[d_model]` (no bias, no affine ‚Äî T5 uses simplified LayerNorm)
- `x * weight` where weight is learned scale (no bias term in T5)

### T5Attention
- Q, K, V projections: `[d_model, d_model]` each
- Reshape to `[B, S, num_heads, head_dim]`
- Attention scores: `q @ k^T / sqrt(head_dim) + relative_position_bias`
- No RoPE, no QK norm
- Dropout on attention weights (0.1)
- Output projection: `[d_model, d_model]`

### T5DenseGatedGELU
- Gate: `wi_0` `[d_ff, d_model]` with `gelu_new` activation
- Up: `wi_1` `[d_ff, d_model]`
- Down: `wo` `[d_model, d_ff]`
- Output: `wo(gelu(wi_0(x)) * wi_1(x))`

### T5Block
```
x ‚Üí LayerNorm ‚Üí SelfAttention(+relative_bias) ‚Üí + x (residual)
  ‚Üí LayerNorm ‚Üí DenseGatedGELU ‚Üí + x (residual)
```

### T5EncoderModel
```
embed(shared.weight[input_ids])  // [B, S, 4096]
for each of 24 blocks:
    x = T5Block(x, relative_bias)
x = final_layer_norm(x)
return x  // [B, S, 4096]
```

### Context Dimension Handling
T5 outputs `[B, S, 4096]` but the transformer's hidden_dim is 2048. Two options:
- **Option A**: Pass `context_dim=4096` to transformer ‚Äî cross-attention will project internally
- **Option B**: Add projection layer 4096‚Üí2048 in the encoder

**Chosen: Option A** ‚Äî the transformer's `TransformerAttention` already handles arbitrary `context_dim` via its Q/K/V projections. The cross-attention K/V projections will learn to map 4096‚Üíhead_dim internally. No extra projection needed.

### Weight Loading
Checkpoint key mapping:
```
encoder.block.{i}.layer.0.SelfAttention.{q,k,v,o}.weight ‚Üí blocks[i].self_attn.{q,k,v,o}_proj.weight
encoder.block.{i}.layer.0.layer_norm.weight ‚Üí blocks[i].input_norm.weight
encoder.block.{i}.layer.1.DenseReluDense.{wi_0,wi_1,wo}.weight ‚Üí blocks[i].ffn.{gate,up,down}_proj.weight
encoder.block.{i}.layer.1.layer_norm.weight ‚Üí blocks[i].post_norm.weight
encoder.block.0.layer.0.SelfAttention.relative_attention_bias.weight ‚Üí relative_bias.weight
encoder.final_layer_norm.weight ‚Üí final_norm.weight
shared.weight ‚Üí embed_tokens.weight
```

The `load_weights` function already does name-matching, so the T5 encoder just needs its VarStore paths to match the checkpoint keys after mapping.

### Tokenizer Reuse
T5 uses SentencePiece (same as Gemma). The existing `LTXVGemmaTokenizer` works for both ‚Äî both use `pad_token_id=0`, `eos_token_id=1`. No changes needed to the tokenizer.

## Implementation Steps

### Step 1: T5Config + T5LayerNorm
- Add `T5ConfigData` to `config.rs`
- Implement `T5LayerNorm` in `t5_encoder.rs` (weight-only LayerNorm)

### Step 2: T5RelativePositionBias
- Implement logarithmic bucketing
- Forward: compute relative positions ‚Üí bucket indices ‚Üí lookup bias table ‚Üí return `[B, H, S, S]`

### Step 3: T5Attention
- Q/K/V projections, reshape, attention with relative position bias, dropout, output projection

### Step 4: T5DenseGatedGELU
- Gate with gelu_new, element-wise multiply, down projection

### Step 5: T5Block + T5EncoderModel
- Compose all pieces: 24 blocks, shared embedding, final norm
- Accept `Path` parameter for weight loading

### Step 6: T5TextEncoder wrapper
- Add to `encoder.rs`: tokenize ‚Üí embed ‚Üí encode ‚Üí return `[1, S, 4096]`
- Add to `configurator.rs`: T5-specific `from_config`

### Step 7: Wire to inference pipeline
- CLI: when `--text-weights` points to T5 checkpoint, use T5 encoder
- GUI: same logic in inference thread
- Auto-detect T5 vs Gemma3 from checkpoint keys (T5 has `encoder.block.0`, Gemma3 has `layers.0`)
