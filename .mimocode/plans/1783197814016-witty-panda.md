# Plan: Minimalist GUI for LTX-R

## Goal
Add a comprehensive GUI application to the LTX-R workspace that lets users interact with the video diffusion pipeline through a clean, minimalist interface.

## Framework Choice: `eframe` (egui)
- Pure Rust, immediate-mode GUI
- ~30KB binary overhead
- No webview, no JS — native rendering via wgpu/glow
- Minimalist by default — clean panels, no chrome
- Cross-platform (Linux/macOS/Windows)

## Architecture

### New Crate: `crates/ltx-app/`

```
crates/ltx-app/
├── Cargo.toml
└── src/
    ├── main.rs          # Entry point, eframe setup
    ├── app.rs           # Top-level App state + update loop
    ├── panels/
    │   ├── mod.rs
    │   ├── sidebar.rs   # Controls panel (left)
    │   ├── viewport.rs  # Frame display (center)
    │   └── status.rs    # Bottom status bar
    ├── state.rs         # Application state (AppConfig, InferenceState)

    └── export.rs        # Save frames/video
```

### State Model

```rust
struct AppState {
    // User inputs
    prompt: String,
    weights_path: Option<PathBuf>,
    height: i64,
    width: i64,
    frames: i64,
    steps: usize,
    cfg_scale: f64,
    scheduler: SchedulerKind,  // Ltx2 | LinearQuadratic | Beta
    device: DeviceKind,         // CPU | CUDA

    // Inference state
    inference: InferenceState,
    frames_display: Vec<frame::Frame>,  // decoded frames for viewport
    current_frame: usize,

    // Output
    output_dir: Option<PathBuf>,
}

enum InferenceState {
    Idle,
    Loading,
    Denoising { step: usize, total: usize, sigma: f64 },
    Decoding,
    Done,
    Error(String),
}
```

### Inference Thread

The denoising loop runs on a `std::thread::spawn` background thread. Communication via `std::sync::mpsc`:

```rust
enum GuiEvent {
    Progress { step: usize, total: usize, sigma: f64 },
    FrameReady(Vec<frame::Frame>),  // decoded frames
    Error(String),
}

enum GuiCommand {
    Start(InferenceParams),
    Cancel,
}
```

## Implementation Steps

### Step 1: Scaffold crate + eframe window
- Create `crates/ltx-app/` with Cargo.toml
- Add to workspace members
- Implement `main.rs` with eframe `run_native`
- Minimal window: 1024×768, title "LTX-R"

### Step 2: Application state
- Define `AppState`, `InferenceState`, `GuiEvent`, `GuiCommand`
- Wire up channels for thread communication
- Default values matching CLI (16×16, 20 steps, CFG 7.5)

### Step 3: Sidebar panel (controls)
- Prompt text input
- Weights file picker (via `rfd`)
- Resolution inputs (height, width, frames)
- Steps slider (1–100)
- CFG scale slider (1.0–20.0)
- Scheduler dropdown (Ltx2, LinearQuadratic, Beta)
- Device selector (CPU, CUDA if available)
- Generate button
- Export button (save frames/video)

### Step 4: Viewport panel
- Display current frame as texture
- Frame navigation (prev/next or scrubber)
- Frame count indicator

### Step 5: Status bar
- Inference state text
- Current step / total
- Current sigma value
- Output dimensions

### Step 6: Background inference
- Spawn inference thread on Generate
- Build model, load weights, run denoising loop
- Send progress events via channel
- Decode frames and send back for display
- Handle cancellation

### Step 7: Export
- Save frames as PNG
- Optional ffmpeg video creation
- Output directory picker
